# YT Pulse

Личная панель мониторинга YouTube-каналов. Она автоматически находит свежие видео по заданным фразам, проверяет заголовки, описания и теги, а затем считает:

- средние просмотры роликов за последние 3 месяца;
- количество роликов, вошедших в расчёт;
- просмотры и комментарии каждого из последних 4 роликов;
- совпавшие ключевые фразы.

Сайт статический и бесплатно размещается на GitHub Pages. Сбор данных запускается раз в сутки через GitHub Actions, поэтому отдельный сервер не нужен, а ключ YouTube API не попадает в браузер.

## Бесплатный адрес

После публикации адрес будет таким:

```text
https://ВАШ-ЛОГИН.github.io/ИМЯ-РЕПОЗИТОРИЯ/
```

Это бесплатный адрес на домене `github.io`. Например, для пользователя `alex` и репозитория `yt-pulse`: `https://alex.github.io/yt-pulse/`.

## Публикация

1. Создайте новый публичный репозиторий на GitHub и загрузите в него все файлы проекта. Основная ветка должна называться `main`.
2. Откройте [Google Cloud Console](https://console.cloud.google.com/), создайте проект и включите **YouTube Data API v3**.
3. В разделе **APIs & Services → Credentials** создайте API key. Рекомендуется ограничить его только API `YouTube Data API v3`.
4. В репозитории GitHub откройте **Settings → Secrets and variables → Actions → New repository secret**.
5. Создайте секрет с именем `YOUTUBE_API_KEY` и вставьте ключ.
6. Откройте **Settings → Pages** и в поле **Source** выберите **GitHub Actions**.
7. Откройте вкладку **Actions**, выберите workflow **Update YouTube statistics and deploy** и нажмите **Run workflow**.

После первого выполнения демо-данные заменятся реальными. Далее обновление будет запускаться ежедневно примерно в 06:17 по Москве; GitHub может запускать плановые задачи с небольшой задержкой.

## Настройка поиска

Фразы и лимиты находятся в [`config/keywords.json`](config/keywords.json).

- `keywords` — поисковые фразы;
- `searchResultsPerKeyword` — сколько свежих результатов проверить для каждой фразы;
- `channelSearchResultsPerKeyword` — сколько каналов проверить по названию;
- `maxChannels` — максимальное число каналов в панели;
- `maxVideosPerChannel` — сколько загрузок канала можно проверить;
- `searchContext` и `contextTerms` — защита от нерелевантных результатов по коротким словам;
- `regionCode` и `relevanceLanguage` — регион и язык поиска.

Сборщик выполняет отдельный поиск видео и каналов для каждой фразы. YouTube ограничивает число поисковых запросов API, поэтому не добавляйте больше 40 фраз при ежедневном запуске.

## Каналы, которые нужно отслеживать всегда

Добавьте ID канала, ссылку вида `/channel/UC...` или `@handle` в [`config/channels.json`](config/channels.json):

```json
{
  "channels": [
    "@example",
    "https://www.youtube.com/channel/UCxxxxxxxxxxxxxxxxxxxxxx"
  ]
}
```

Такие каналы попадут в сбор независимо от результатов автопоиска.

## Локальный просмотр

На Windows:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\serve.ps1
```

Затем откройте `http://localhost:4173/`.

Для ручного обновления данных нужен Node.js 20+:

```powershell
$env:YOUTUBE_API_KEY = "ваш-ключ"
npm run update
```

## Важные особенности

- Статистика просмотров и комментариев берётся из публичных полей YouTube Data API.
- Если комментарии у видео отключены, API может не вернуть `commentCount`; панель показывает `0`.
- Среднее считается по доступным публичным видео, опубликованным за последние 3 календарных месяца.
- GitHub Pages публикует сайт открыто. Не добавляйте в JSON или код личные секреты и API-ключи.
